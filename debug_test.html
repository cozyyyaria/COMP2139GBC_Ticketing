Event Creation Debug Test

Step 1: Check Application Status

Application should be running at: http://localhost:5128

Events page: http://localhost:5128/Event

Step 2: Test Event Creation

Go to: http://localhost:5128/Event/Create

Fill out the form with these exact values:

  • Title: Summer Concert 2025
  • Event Date: 2025-06-15T19:00
  • Category: Select any available category
  • Ticket Price: 25.99
  • Available Tickets: 100

Step 3: Check for Errors

After clicking "Save", look for:

  • Red error messages at the top of the form
  • Red error messages under specific fields
  • Browser console errors (press F12)

Step 4: Verify Success

If successful, you should be redirected to: http://localhost:5128/Event

Your event should appear in the list with search/filter functionality.

Common Issues & Solutions

  • Form validation errors: Make sure all fields are filled correctly
  • Category not selected: Choose a category from the dropdown
  • Date format issues: Use the datetime-local format (YYYY-MM-DDTHH:MM)
  • Price format: Use decimal format (25.99, not 25,99)

Expected Result

After successful creation, you should see:

  • Redirect to Events list page
  • Your event appears in the table
  • Search and filter controls work
  • Overview page shows event count
